## Welcome

I’m **0xh4ty**. I work in security and spend most of my time understanding how systems break and why.

This blog documents what I learn, from web application vulnerabilities to reverse engineering Linux binaries and exploring blockchain systems.

## How I Got Here

I started in Web2 security and spent two years building fundamentals in penetration testing and web exploitation. During this time, I earned the HTB Certified Web Exploitation Specialist (CWES) certification and developed a strong grounding in vulnerability classes and exploitation patterns.

I am currently pursuing a Master of Computer Applications (MCA) while continuing to deepen my focus on systems and security research.

Over time, I became more interested in how systems work internally. That led me toward blockchain technologies, Rust, and lower-level program analysis.

## What I Work On

My recent work includes:

- Reproducing real-world smart contract vulnerabilities
- Writing exploit proof-of-concepts
- Reverse engineering C++ binaries using Ghidra and GDB
- Contributing to compiler-adjacent Rust tooling
- Building a Solidity language server for Emacs
- Building the Rust-based static site generator that powers this blog

I prefer understanding systems deeply rather than treating them as black boxes.

## Tooling

Linux is my primary environment and Rust is my main language. Emacs is my primary editor and workflow hub.

## Current Focus

I am currently focused on reverse engineering, vulnerability discovery, and strengthening low-level systems intuition through Rust and C++.
